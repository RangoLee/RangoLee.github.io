C++ 中new和new()的区别

结论：new()调用没有参数的构造函数，new调用默认构造函数或唯一构造函数。
C++在new时的初始化规律可能为：对于有构造函数的类，new和new()都调用构造函数进行初始化；
如果没有构造函数，则new只分配内存空间，不初始化；new()分配内心空间的同时初始化。

STL模板技巧

发表于 2016-12-02   |   分类于 C++   |     |   阅读次数

typename

声明模板参数：template<typename T> ...,只有在这种语境下，typename关键字才可以和class关键字互换

  • class关键字
    声明类：class A{...};
    声明模板参数：template<class T> ...
  • typename关键字
    声明模板参数：template<typename T> ...
    解决嵌套依赖：typename T::A a;
  • 无论是声明模板参数还是解决嵌套依赖，都不能使用struct关键字

解决嵌套依赖

  • 在第一次编译模板代码时，模板参数的具体类型尚不明确，编译器会把依赖于模板参数的嵌套类型理解为某个类的静态成员变量，因此当它看到代码中使用这样的标识符声明其它变量时，会报错误，这就叫嵌套依赖。

  • typename关键字旨在告诉编译器，所引用的标识符是个类型名，可以声明变量，具体类型等到实例化时再定
    template<typename T> void foo(){typename T::A a; ....}
